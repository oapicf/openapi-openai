createImageEdit

URL

https://api.openai.com/v1/images/edits

HTTP Method

POST

Response Type

ImagesResponse

Parameters

  • Form: image File(file)

    The image to edit. Must be a valid PNG file, less than 4MB, and square. If mask is not provided, image must have transparency, which will be used as the mask.

  • Form: prompt String(string)

    A text description of the desired image(s). The maximum length is 1000 characters.

  • Form: mask File(file)

    An additional image whose fully transparent areas (e.g. where alpha is zero) indicate where `image` should be edited. Must be a valid PNG file, less than 4MB, and have the same dimensions as `image`.

  • Form: n Integer(integer)

    The number of images to generate. Must be between 1 and 10.

  • Form: size String(string)

    The size of the generated images. Must be one of `256x256`, `512x512`, or `1024x1024`.

  • Form: response_format String(string)

    The format in which the generated images are returned. Must be one of `url` or `b64_json`.

  • Form: user String(string)

    A unique identifier representing your end-user, which can help OpenAI to monitor and detect abuse. [Learn more](/docs/guides/safety-best-practices/end-user-ids).

createTranscription

URL

https://api.openai.com/v1/audio/transcriptions

HTTP Method

POST

Response Type

CreateTranscriptionResponse

Parameters

  • Form: file File(file)

    The audio file object (not file name) to transcribe, in one of these formats: mp3, mp4, mpeg, mpga, m4a, wav, or webm.

  • Form: model CreateTranscriptionRequest_model(CreateTranscriptionRequest_model)

  • Form: prompt String(string)

    An optional text to guide the model's style or continue a previous audio segment. The [prompt](/docs/guides/speech-to-text/prompting) should match the audio language.

  • Form: response_format String(string)

    The format of the transcript output, in one of these options: json, text, srt, verbose_json, or vtt.

  • Form: temperature BigDecimal(number)

    The sampling temperature, between 0 and 1. Higher values like 0.8 will make the output more random, while lower values like 0.2 will make it more focused and deterministic. If set to 0, the model will use [log probability](https://en.wikipedia.org/wiki/Log_probability) to automatically increase the temperature until certain thresholds are hit.

  • Form: language String(string)

    The language of the input audio. Supplying the input language in [ISO-639-1](https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes) format will improve accuracy and latency.

listFineTuneEvents

URL

https://api.openai.com/v1/fine-tunes/{fine_tune_id}/events

HTTP Method

GET

Response Type

ListFineTuneEventsResponse

Parameters

  • Path: fine_tune_id String

    The ID of the fine-tune job to get events for.

  • Query: stream Boolean

    Whether to stream events for the fine-tune job. If set to true, events will be sent as data-only [server-sent events](https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events/Using_server-sent_events#Event_stream_format) as they become available. The stream will terminate with a `data: [DONE]` message when the job is finished (succeeded, cancelled, or failed). If set to false, only events generated so far will be returned.
