post /audio/transcriptions
Transcribes audio into the input language. (createTranscription)

Consumes

This API call consumes the following media types via the Content-Type request header:
  • multipart/form-data

Form parameters

file (required)
Form Parameter — The audio file object (not file name) to transcribe, in one of these formats: mp3, mp4, mpeg, mpga, m4a, wav, or webm. default: null format: binary
model (required)
Form Parameter — default: null
prompt (optional)
Form Parameter — An optional text to guide the model's style or continue a previous audio segment. The prompt should match the audio language. default: null
response_format (optional)
Form Parameter — The format of the transcript output, in one of these options: json, text, srt, verbose_json, or vtt. default: json
temperature (optional)
Form Parameter — The sampling temperature, between 0 and 1. Higher values like 0.8 will make the output more random, while lower values like 0.2 will make it more focused and deterministic. If set to 0, the model will use log probability to automatically increase the temperature until certain thresholds are hit. default: 0
language (optional)
Form Parameter — The language of the input audio. Supplying the input language in ISO-639-1 format will improve accuracy and latency. default: null

Return type

CreateTranscriptionResponse

Example data

Content-Type: application/json
{
  "text" : "text"
}

Path parameters

fine_tune_id (required)
Path Parameter — The ID of the fine-tune job to get events for. default: null

Query parameters

stream (optional)
Query Parameter —

Whether to stream events for the fine-tune job. If set to true, events will be sent as data-only server-sent events as they become available. The stream will terminate with a data: [DONE] message when the job is finished (succeeded, cancelled, or failed).

If set to false, only events generated so far will be returned.

default: false

By default, the learning rate multiplier is the 0.05, 0.1, or 0.2 depending on final batch_size (larger learning rates tend to perform better with larger batch sizes). We recommend experimenting with values in the range 0.02 to 0.2 to see what produces the best results.

prompt_loss_weight (optional)
BigDecimal

The weight to use for loss on the prompt tokens. This controls how much the model tries to learn to generate the prompt (as compared to the completion which always has a weight of 1.0), and can add a stabilizing effect to training when completions are short.

If prompts are extremely long (relative to completions), it may make sense to reduce this weight so as to avoid over-prioritizing learning the prompt.

compute_classification_metrics (optional)
Boolean

If set, we calculate classification-specific metrics such as accuracy and F-1 score using the validation set at the end of every epoch. These metrics can be viewed in the results file.

In order to compute classification metrics, you must provide a validation_file. Additionally, you must specify classification_n_classes for multiclass classification or classification_positive_class for binary classification.

classification_n_classes (optional)
Integer

The number of classes in a classification task.

This parameter is required for multiclass classification.

classification_positive_class (optional)
String

The positive class in binary classification.

This parameter is needed to generate precision, recall, and F1 metrics when doing binary classification.

classification_betas (optional)
array[BigDecimal]

If this is provided, we calculate F-beta scores at the specified beta values. The F-beta score is a generalization of F-1 score. This is only used for binary classification.

With a beta of 1 (i.e. the F-1 score), precision and recall are given the same weight. A larger beta score puts more weight on recall and less on precision. A smaller beta score puts more weight on precision and less on recall.

suffix (optional)
String

A string of up to 40 characters that will be added to your fine-tuned model name.

For example, a suffix of "custom-model-name" would produce a model name like ada:ft-your-org:custom-model-name-2022-02-15-04-21-04.
